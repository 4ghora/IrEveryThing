## 1. Alert Name

**Potential Ransomware Activity**

---

## 2. Alert Description (Detection Logic Explanation)

### 2.1 What Triggered the Alert

This alert is typically triggered by **behavioral anomaly detection** and/or **model deviation** at the network flow and protocol level. In Darktrace, detection is usually AI-based rather than signature-only.

Common triggers include:

* **Unusual SMB file write volume** (burst of encrypted file writes)
* **High-rate file renames with random extensions** over SMB
* **Abnormal internal scanning + SMB session establishment**
* **Outbound connections to rare external IPs/domains before encryption**
* **Beacon-like C2 traffic over HTTP/HTTPS/DNS**
* **Large internal lateral SMB/RPC traffic spike**
* **Sudden increase in entropy in file transfers (encrypted data patterns)**

Detection can occur at:

* **Packet level**: SMB Write requests, abnormal SMB flags
* **Flow level**: Spike in connections, unusual connection frequency
* **Payload level**: High entropy, encrypted payload anomalies, protocol misuse

Example:

* Host normally performs ~50 SMB writes/hour.
* Suddenly performs 20,000 writes in 10 minutes.
* Writes files with extensions like `.lockbit`, `.conti`, `.encrypted`.
* Initiates connections to 5 new internal servers via TCP 445.

This deviation triggers the alert.

---

### 2.2 Security Tool

Alert generated by:

* Darktrace
  AI-driven Network Detection and Response (NDR) platform using self-learning behavioral analytics.

---

### 2.3 Why This Alert is Important

From a **network security perspective**, ransomware:

* Propagates laterally using SMB, RDP, RPC
* May exfiltrate data before encryption (double extortion)
* Establishes Command & Control (C2)
* Can encrypt file servers, domain controllers, backups
* Causes rapid enterprise-wide impact

Network-level detection is often **the earliest visibility point** before encryption completes.

---

### 2.4 Define Severity

**Default Severity: High**

Escalate to **Critical** if:

* Encryption activity confirmed
* Domain Controller involved
* Backup server targeted
* Known ransomware C2 IP/domain observed
* Data exfiltration confirmed
* Multiple hosts impacted within minutes
* Activity in PCI/CDE or production server subnet

## 3. Knowledge Required Before Investigation

This section must be clearly understood before proceeding.

---

### 3.1 Core Network & Protocol Concepts

#### A. SMB (Server Message Block)

* Runs over TCP 445
* Used for file sharing, network drives, printer sharing
* Common for ransomware propagation

Important SMB operations:

* **SMB2 WRITE**
* **SMB2 CREATE**
* **SMB2 SET_INFO (rename)**
* **Tree Connect**
* **Session Setup**

Ransomware behavior:

* Opens file
* Reads contents
* Encrypts locally
* Writes encrypted version
* Renames file
* Deletes original

Network Indicator:

* Massive increase in SMB2 WRITE operations
* File rename bursts
* High file entropy transfers

---

#### B. Lateral Movement Protocols

Common network protocols attackers use:

* SMB (445)
* RDP (3389)
* RPC (135)
* WinRM (5985/5986)
* WMI over DCOM
* PsExec (SMB + service creation)

Network Indicators:

* One host connecting to many internal systems rapidly
* Failed SMB session attempts
* NTLM authentication bursts

---

#### C. Command & Control (C2)

Ransomware often communicates with C2 before encryption.

Techniques:

* HTTP/HTTPS beaconing
* DNS tunneling
* Fast flux infrastructure
* Rare external domains

Network Indicators:

* Regular periodic outbound connections
* Small, consistent packet sizes
* High DNS query entropy
* Rare ASN or geolocation

---

#### D. Data Exfiltration (Double Extortion)

MITRE ATT&CK Network Techniques:

* T1041: Exfiltration Over C2 Channel
* T1048: Exfiltration Over Alternative Protocol
* T1021: Remote Services (Lateral Movement)

Indicators:

* Large outbound uploads
* Compression before exfil
* Traffic to cloud storage or unknown VPS

---

#### E. Traffic Entropy & Encryption Indicators

Encrypted data exhibits:

* High entropy
* Uniform byte distribution
* Non-compressible data

If internal SMB writes suddenly show encrypted-like entropy, ransomware is likely.

---

#### F. Network Segmentation Awareness

Analyst must understand:

* Where Domain Controllers reside
* Where file servers reside
* Backup network zones
* Critical production VLANs
* DMZ boundaries

Lateral movement across segments = severe compromise.

## 4. Attacker Perspective

### 4.1 Why Attackers Use This Network Technique

Ransomware operators:

* Move laterally fast
* Encrypt network shares
* Target backup systems
* Destroy recovery capability
* Maximize business impact

Network-based propagation is faster than endpoint-only encryption.

---

### 4.2 What They Try to Achieve

* Privilege escalation
* Lateral movement
* Disable backups
* Exfiltrate sensitive data
* Encrypt file servers
* Domain-wide impact

---

### 4.3 Real-World Examples

* Conti – aggressive SMB lateral movement
* LockBit – automated domain propagation
* WannaCry – wormable SMB exploitation (EternalBlue)

These leveraged network spread mechanisms heavily.

---

### 4.4 Potential Business Impact

* Full operational shutdown
* Production halt
* Regulatory penalties
* Data breach notification
* Reputational damage
* Multi-million-dollar losses

## 5. Pre-Investigation Checklist

### 5.1 Confirm:

* Source IP
* Destination IP(s)
* Ports (445, 3389, 135)
* MAC address
* DHCP mapping
* Username (if identity integration exists)

---

### 5.2 Check:

* Network segment (Server farm? Domain controller VLAN?)
* Direction:

  * Internal → Internal (lateral)
  * Internal → External (C2 or exfil)
* Critical asset involvement

---

### 5.3 Verify:

* Alert severity
* Concurrent alerts:

  * SMB anomalies
  * C2 detection
  * Rare external connections
  * Data exfil alerts
* Same source IP triggering multiple models

---

## 6. Investigation Steps

---

### 6.1 Questions Analyst Must Ask

1. Is this host normally a file server or workstation?
2. Has SMB traffic volume significantly deviated from baseline?
3. Are multiple internal hosts being accessed?
4. Is there evidence of external C2 communication?
5. Is data being exfiltrated?
6. Are file servers or backups being targeted?
7. Is this automated or user-driven activity?
8. Is encryption confirmed at endpoint level?

---

### 6.2 Answers to 6.1

1. If workstation → suspicious lateral movement.
2. If 10x–100x SMB spike → likely ransomware.
3. If many hosts contacted sequentially → worm-like spread.
4. Rare external IP + beaconing → C2 established.
5. Large outbound transfer → double extortion.
6. Backup server touched → escalate immediately.
7. High-speed automation → malware.
8. Confirm with EDR logs.

---

### 6.3 Major Investigations

* Analyze SMB session logs (WRITE/RENAME counts)
* Check Zeek/Corelight logs if available
* Inspect flow volume spikes
* Review DNS logs for rare domains
* Check TLS fingerprinting (JA3/JA4)
* Identify authentication pattern anomalies
* Check for service creation events over SMB
* Review internal scan patterns
* Correlate with EDR encryption events

---

### 6.4 Minor Investigations

* User login anomalies
* VPN login correlation
* Recent privilege escalation
* Admin credential usage
* PowerShell remoting traffic
* Internal RPC burst activity

---

## 7. Evidence to Collect

* PCAP (if available)
* NetFlow records
* SMB logs
* DNS logs
* Proxy logs
* Firewall logs
* EDR telemetry
* Host timeline
* Authentication logs (AD)
* File server audit logs

---

## 8. Indicators of True Positive

* Mass SMB writes
* File rename burst
* High entropy transfers
* Rare outbound C2
* Lateral movement to many hosts
* Backup server access
* EDR confirms encryption
* Domain admin account misuse

---

## 9. Indicators of False Positive

* Legitimate bulk file migration
* Backup job execution
* Software deployment
* Patch management tools
* File server maintenance
* Storage replication event

Validate with change management logs.

---

## 10. Incident Response Actions (If True Positive)

---

### 10.1 Containment

* Immediately isolate host (NAC or EDR)
* Block outbound C2 IP/domain
* Disable compromised accounts
* Segregate impacted VLAN
* Block SMB laterally if possible

---

### 10.2 Eradication

* Identify patient zero
* Remove malware
* Reset credentials
* Patch exploited vulnerabilities
* Remove persistence mechanisms

---

### 10.3 Recovery

* Restore from clean backups
* Verify backup integrity
* Monitor for reinfection
* Reintroduce hosts gradually
* Increase logging temporarily

---

## 11. Mitigation & Prevention

* SMB network segmentation
* Disable SMBv1
* Enforce least privilege
* EDR with ransomware rollback
* DNS filtering
* Strict outbound firewall rules
* MFA for admin accounts
* Regular backup isolation
* Lateral movement detection rules
* Disable NTLM where possible

---

## 12. Actions an IR Should Never Do (In Context of Alert)

* Do NOT immediately reboot infected host
* Do NOT delete encrypted files prematurely
* Do NOT notify attacker
* Do NOT restore backups before full containment
* Do NOT disable logging
* Do NOT ignore internal-only spread
* Do NOT assume “no external traffic = safe”
